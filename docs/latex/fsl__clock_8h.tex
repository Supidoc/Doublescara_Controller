\doxysubsubsection*{Driver version}
Configures whether to check a parameter in a function.

Some MCG settings must be changed with conditions, for example:\+
\begin{DoxyEnumerate}
\item MCGIRCLK settings, such as the source, divider, and the trim value should not change when MCGIRCLK is used as a system clock source.
\item MCG\+\_\+\+C7\+[OSCSEL]\+ should not be changed when the external reference clock is used as a system clock source. For example, in FBE/\+\+BLPE/\+\+PBE modes.
\item The users should only switch between the supported clock modes.
\end{DoxyEnumerate}

MCG functions check the parameter and MCG status before setting, if not allowed to change, the functions return error. The parameter checking increases code size, if code size is a critical requirement, change \+\#\+MCG\+\_\+\+CONFIG\+\_\+\+CHECK\+\_\+\+PARAM to 0 to disable parameter checking.

When set to 0, peripheral drivers will enable clock in initialize function and disable clock in de-\/initialize function. When set to 1, peripheral driver will not control the clock, application could control the clock out of the driver.

\begin{DoxyNote}{Note}
All drivers share this feature switcher. If it is set to 1, application should handle clock enable and disable for all drivers. 
\end{DoxyNote}
