Retrieves the stepper motor handle by its label identifier.

This function searches for a stepper motor by its label and returns a pointer to its handle structure if found.

Parameters
[in]labelThe label identifier of the stepper motor to find.
[out]handlePointer to store the found stepper motor handle.
Note
This function is TASK-SAFE and can be called from any task context.
The label must match exactly as configured during STP_init_stepper().
Returns
kStatus_Success if the stepper motor is found. kStatus_Fail if the stepper motor with the given label is not found.

◆ STP_init()

status_t STP_init ( void )

Initializes the stepper motor control module.

This function initializes the movement and configuration queues used for task-safe communication. It must be called before using any other functions in this module.

Note
This function is NOT task-safe and must be called during system initialization.
The STP_task() should be started after this function completes successfully.
Ensure FTM timer and GPIO peripherals are initialized before calling this.
Returns
kStatus_Success if initialization is successful. kStatus_Fail if queue creation fails.

STP_task()

Initializes a stepper motor with the provided configuration.

This function sends a configuration command to the stepper control task. The stepper motor will be configured according to the provided parameters.

Parameters
[in]configConfiguration structure containing stepper parameters.
Note
This function is TASK-SAFE and can be called after STP_init().
Configuration is applied asynchronously by the STP_task().
Warning
Do not modify the config structure after calling this function.
Returns
kStatus_Success if the configuration is successfully queued. kStatus_Fail if the queue is full or an error occurs.

◆ STP_move_relative()

status_t STP_move_relative ( STP_StepperHandle_t * handle,
uint32_t stepCount,
STP_Direction_t direction )

Sends a relative movement command to a stepper motor.

This function adds a movement command to the queue for the specified stepper. The movement will be executed asynchronously by the STP_task function.

Parameters
[in]handlePointer to the stepper motor handle.
[in]stepCountNumber of steps to move.
[in]directionDirection of movement (clockwise or counterclockwise).
Note
This function is TASK-SAFE and can be called from any task context.
The actual movement occurs in the STP_task() and is non-blocking.
Warning
Multiple calls will queue movements to execute sequentially.
Returns
kStatus_Success if the command is successfully queued. kStatus_Fail if the queue is full or an error occurs.

◆ STP_stop()

status_t STP_stop ( STP_StepperHandle_t * handle,
uint8_t doDeceleration )

Stops a stepper motor movement.

This function sends a command to stop the specified stepper motor. The motor can be stopped immediately or with a deceleration phase to smoothly bring it to rest, depending on the doDeceleration parameter.

Parameters
[in]handlePointer to the stepper motor handle.
[in]doDecelerationIf 1, the motor will decelerate to a stop. If 0, the motor will stop immediately.
Note
This function is TASK-SAFE and can be called from any task context.
The stop command is queued and executed by the STP_task().
Returns
kStatus_Success if the stop command is successfully queued. kStatus_Fail if the queue is full or an error occurs.
See also
STP_move_relative()

◆ STP_task()

void STP_task ( void * pvParameters)

Task function for processing stepper motor commands.

This function is intended to be run as a FreeRTOS task. It continuously processes configuration and movement commands from the queues and manages the stepper motor state machine.

Parameters
[in]pvParametersPointer to task parameters (not used in this implementation).
Note
This task should be started after STP_init() completes successfully.
This task is blocked on command queues and wakes when commands are available.
Warning
This function does not return; it is expected to run as a perpetual FreeRTOS task.

STP_init()
